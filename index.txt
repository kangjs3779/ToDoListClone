자바스크립트라는 언어가 있음
html 정적인 페이지를 만드는 언어
javascript 정적인 html 파일을 동적으로 만들어줌(html 조작)

javascript 해석은 누가하나? 브라우저가 함
브라우저마다 쓰는 해석 엔진들이 다 다름
크롬 : v8
모질라 : spiderMonkey

크롬에서 v8에 살을 붙여서 나온 애가 node.js임
브아루저 내에서 말고도 다른 환경에서도 jacascript를 실행할 수 있도록함
node 덕분에 jacascript를 프로그래밍 언어처럼 사용함
서버도 가능

왜 하필 node로?
노드로 만든 서버 / 일반 언어로 만든 서버
일반 언어로 하면 중간에 버거운 요청을 받으면 다른 요청이 연기됨
노드는 ? 요청을 일단 다 받고 처리속도가 빠른 것부터 처리를 한다, 순서는 상관없음
이런 작업을 Non-blocking I/O라고 한다

이런 특징은 SNS나 채팅서비스에서 요청이 아주 많을 때, 무거운 요청이 있어도 요청 대기 시간이 없다
물론 일반 언어로도 가능함

이러한 장점때문에 많이 쓰임
코드가 매우 짧고 쉬움 -> 빠른 개발 가능함
근데 이미지 처리 서버가 필요할 때에는 좀 안좋음
통계/분석 또한 파이썬 같은게 좋음

express라는 라이브러리를 써서 서버를 만들 것임
코드를 쉽게 짜기 위해 빌려쓰는 코드 모음집 : 라이브러리

터미널 킴
1) npm init
npm은 라이버리 설치를 도와주는 도구임
어떤 라이브러리를 설치했는지 기록하면 좋을 듯?
package.json에 기록함

2) npm install express
설치할 때마다 package.json에 기록이 된다

설치하고 나면 node_module이 생긴 것을 확인 할 수 있다

npm이 안되면 yarn으로 할 수 있음
명령어가 다름
yarn add express

노드를 재실행하기 귀찮으니까 자동화시킴
npm install -g nodemon
글로벌하게 설치해달라
yarn으로도 할 수 있음
